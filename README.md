# ATB List

Веб-застосунок для створення списків покупок з автоматичним пошуком товарів у мережі супермаркетів АТБ.

## Що це таке і як це працює

Користувач створює список покупок, вводячи назви потрібних продуктів через кому або з нового рядка. Після створення система дозволяє для кожної позиції знайти відповідний товар в АТБ — достатньо натиснути кнопку пошуку поруч із позицією. Застосунок виконує запит до сайту atbmarket.com, парсить результати та показує доступні варіанти з назвою, фото та ціною. Користувач обирає потрібний товар, і він зберігається в списку.

Щоб не навантажувати сайт АТБ повторними запитами, результати пошуку кешуються в базі даних на 2 години. Якщо той самий запит вже виконувався нещодавно — відповідь повертається з кешу миттєво.

Під час походу в магазин зручно відмічати куплені товари прямо в застосунку. Прогрес-бар на картці списку і на сторінці деталі показує скільки позицій вже у кошику. Готові списки можна дублювати — наприклад, якщо щотижня купуєте приблизно одне й те саме.

Всі дані зберігаються локально у файловій базі H2, без реєстрації та авторизації.

## Технічний стек

**Backend**
- Java 17
- Spring Boot 4.x
- Spring Web MVC — REST API та серверний рендеринг сторінок
- Spring Data JPA / Hibernate — робота з базою даних
- Jsoup — парсинг HTML-сторінок atbmarket.com
- Jackson 3 (tools.jackson) — серіалізація кешу пошуку в JSON

**База даних**
- H2 Database (файловий режим) — зберігає списки покупок та кеш пошукових запитів

**Frontend**
- Thymeleaf — серверна шаблонізація
- Vanilla JS — динамічні дії без перезавантаження сторінки (вибір товару, відмітка куплено, перейменування)
- CSS Custom Properties — єдина дизайн-система з перевикористовуваними компонентами

**Структура бази даних**

| Таблиця | Призначення |
|---|---|
| `shopping_lists` | Списки покупок |
| `shopping_list_items` | Позиції списку з обраними товарами |
| `search_cache` | Кеш результатів пошуку по запиту |
